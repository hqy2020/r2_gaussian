# CoR-GS 技术路线放弃声明

**日期**: 2025-11-26
**决策类型**: 技术路线终止
**状态**: 已放弃

---

## 摘要

经过为期 9 天的深入研究和多轮实验验证（2025-11-17 至 2025-11-26），我们正式宣布**放弃 CoR-GS (Co-Regularization Gaussian Splatting) 技术路线**。实验结果表明，CoR-GS 的核心创新点无法在 R²-Gaussian CT 重建场景中产生正向收益，所有实验变体的性能均低于 baseline。

## 失败原因分析

### 1. 场景本质差异

| 对比维度 | CoR-GS 原论文场景 | R²-Gaussian CT 场景 |
|----------|------------------|---------------------|
| **视角数量** | 数百张多视角图像 | 仅 3/6/9 个投影视角 |
| **投影方式** | 透视投影 (Perspective) | 正交/锥形 X 射线投影 |
| **几何信息** | 丰富的多视角几何约束 | 极度稀疏的几何约束 |
| **训练视角** | 大量视角提供冗余信息 | 每个视角都至关重要 |

**核心问题**: CoR-GS 的 Pseudo-view Co-regularization 依赖于"视角间的一致性冗余"来提供正则化。但在 3-views 极端稀疏场景中，这种冗余几乎不存在。

### 2. Co-pruning 无效

从训练日志可以看到：

```
[Co-pruning] Model 0: 59046 -> 59046 (pruned 0, 0.0%)
[Co-pruning] Model 1: 63835 -> 63835 (pruned 0, 0.0%)
```

**发现**: 在 CoR-GS Full 实验中，Co-pruning 几乎没有剪除任何高斯点（0%），说明：
- 两个模型学到了几乎相同的表示
- 没有产生论文所期望的"互补视角"效果

当强制使用更激进的 Co-pruning (v2, threshold=0.02) 时：
```
[Co-pruning] Model 0: 13375 -> 12627 (pruned 748, 5.6%)
[Co-pruning] Model 1: 14846 -> 13825 (pruned 1021, 6.9%)
```

虽然剪枝了约 5-7% 的点，但性能反而下降了 0.5 dB，说明被剪掉的点实际上是有用的。

### 3. Pseudo-view Co-regularization 损失极低

```
[CoR-GS] Iter 15000: loss_m0=0.000000, loss_m1=0.000000, scale=1.000
[CoR-GS] Iter 20000: loss_m0=0.000164, loss_m1=0.000164, scale=1.000
[CoR-GS] Iter 25000: loss_m0=0.000000, loss_m1=0.000000, scale=1.000
```

**发现**: Pseudo-view co-regularization 损失值极低（接近 0），说明：
- 两个模型在伪视角上已经高度一致
- 正则化项无法提供有意义的梯度信号
- 即使有梯度，权重也太小无法影响优化

### 4. 双模型训练无法提供互补信息

在 3-views 场景中：
- 两个模型从相同的 3 个视角学习
- 相同初始化 → 相似的优化轨迹
- 差异化初始化 (v2) 也无法创造本质差异

这与原论文的预期相反。原论文中，双模型可以从不同视角子集学习，产生互补信息。但在我们的场景中，所有模型看到的是完全相同的 3 个视角。

## 经验教训

### 必须避免的错误

1. **假设技术通用性**: CoR-GS 在多视角场景的成功不能自动迁移到极端稀疏场景
2. **忽视场景本质差异**: 3 views vs 数百 views 是本质性差异，而非量级差异
3. **过度信任论文声明**: "适用于稀疏场景"的描述可能有特定前提条件

### 有价值的收获

1. **多模型训练架构**: 已实现可复用的双模型训练框架
2. **Co-pruning 实现**: 可用于其他需要模型协同剪枝的场景
3. **场景分析框架**: 建立了评估技术可迁移性的分析方法

## 后续方向建议

既然 CoR-GS 路线已证明无效，建议探索以下替代方向：

1. **BINO (Binocular)**: 已有初步实验，利用双目视角约束
2. **GR (Gaussian Regularization)**: 直接正则化高斯参数
3. **IPSM (Iterative Point-based Supervision)**: 迭代点云监督
4. **数据增强方法**: 专门为极端稀疏场景设计的增强策略
