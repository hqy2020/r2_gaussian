# Bino (双目一致性) 超参数分析报告

> 更新时间: 2025-12-03
> 基于消融实验 (48组) 和超参数搜索实验

## 1. 核心结论

### ⚠️ Bino 单独使用不可靠

| 场景 | Bino-only PSNR | Baseline PSNR | Δ | 状态 |
|------|---------------|---------------|---|------|
| Foot-3 | 28.5234 | 28.7314 | -0.21 | ✗ |
| Chest-6 | 33.3874 | 33.4392 | -0.05 | ✗ |
| Abdomen-9 | 36.9465 | 36.9478 | -0.00 | ✗ |

**消融实验证明：Bino 单独使用在所有场景都无法稳定超越 baseline！**

### ✓ 推荐策略：Bino 必须与其他技术组合使用

最佳组合为 **IXB (Init-PCD + X²-Gaussian + Bino)**：
- Foot-3: PSNR 28.9381 (+0.45 dB) ✓
- 稳定性显著提升

4. `start_iter=5000` 在所有测试场景都超越 baseline

## 4. 推荐配置

### 稳定超越 Baseline 的配置

```bash
# 推荐超参数 (基于搜索最佳结果)
--enable_binocular_consistency \
--binocular_loss_weight 0.08 \        # 降低 (原0.15)
--binocular_max_angle_offset 0.04 \   # 降低 (原0.06, ≈2.3°)
```

## 5. 使用建议

### ✓ 推荐场景
- **Foot-3 views**: 与 IX 组合使用 (IXB)
- **Abdomen-9 views**: 与 X 组合使用 (XB)

### ✗ 不推荐场景
- **Chest-6 views**: 不要使用 Bino，直接用 XF 组合

### 最佳实践

1. **永远不要单独使用 Bino** - 必须与 Init-PCD (I) 或 X²-Gaussian (X) 组合
2. **避免 Bino + FSGS (FB) 组合** - 存在技术冲突，性能下降
3. **使用更保守的超参数** - 小 loss_weight, 小 angle_offset, 早启动
4. **Chest 场景慎用** - Bino 在 6-views 场景效果不佳

## 7. 技术原理

Bino (双目一致性损失) 通过以下机制工作：

1. **虚拟视角生成**: 在训练视角附近生成微小偏移的虚拟视角
2. **双目 Warping**: 利用渲染深度将图像 warp 到虚拟视角
3. **一致性损失**: 约束 warp 后图像与直接渲染图像一致

### 失效原因分析

Bino 在某些场景失效的可能原因：
1. **稀疏视角下深度不准**: 3-views 深度估计噪声大，warp 质量差
2. **与 FSGS 冲突**: 两者都依赖深度，可能产生矛盾梯度
3. **Chest 解剖复杂度高**: 肺部空腔和骨骼交错，几何约束难以生效
