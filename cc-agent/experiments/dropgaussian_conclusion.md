# DropGaussian 技术评估报告

**评估日期**: 2025-11-25
**评估人**: R²-Gaussian 研究团队
**结论**: 放弃 DropGaussian 技术路线

---

## 1. 背景

DropGaussian 是一种针对 3D Gaussian Splatting 的正则化技术，核心思想是在训练过程中随机丢弃部分 Gaussian 点，防止模型过拟合到特定视角。

**官方公式**:
```
drop_rate = gamma * min(iteration / 10000, 1.0)
```
其中 `gamma = 0.2`，即最大丢弃率为 20%。

## 3. 失败原因分析

### 3.1 根本矛盾：信息量不足

DropGaussian 的设计假设是"训练数据充足，需要防止过拟合"。但在 3 views CT 重建场景：

- **训练视角极度稀疏**: 仅 3 个投影角度
- **信息本就不足**: 每个视角都携带不可或缺的几何信息
- **Drop 即损失**: 随机丢弃任何视角都会造成严重信息缺失

### 3.2 中等质量视角损失严重

通过逐视角分析发现：

| 视角质量 | Baseline PSNR 范围 | DropGaussian 影响 |
|---------|-------------------|------------------|
| 低质量 | < 25 dB | +0.43 dB (受益) |
| **中等质量** | 25-30 dB | **-1.02 dB (严重损失)** |
| 高质量 | ≥ 30 dB | +0.11 dB (受益) |

3 views 场景的主力视角恰好落在中等质量区间（25-30 dB），这解释了为什么整体性能下降。

### 3.3 改进尝试及效果

| 改进策略 | 思路 | 效果 |
|---------|------|------|
| Curriculum Learning | 逐步增加 drop rate | -0.15 dB，无效 |
| Importance Aware | 根据 Gaussian 重要性选择性 drop | -0.28 dB，更差 |
| 官方参数对齐 | 严格按论文实现 | -0.41 dB，最差 |
| 视角感知 + Warmup | 距离训练视角越远 drop 越少 | -0.23 dB，仍负 |

所有改进策略都无法使 3 views 场景超越 baseline。

## 5. 结论与决策

### 5.1 放弃理由

1. **核心场景无效**: 3 views 是项目目标场景，所有实验均为负效果
2. **改进空间有限**: 已尝试 4 种改进策略，均无法突破
3. **原理不适配**: DropGaussian 的正则化假设与稀疏视角场景矛盾
4. **投入产出比低**: 6 views 仅 +0.05 dB，不值得继续投入

### 5.2 后续方向

- **专注 X²-Gaussian**: K-Planes 时空特征编码更适合稀疏场景
- **探索其他正则化**: 如 TV Loss、Lipschitz 约束等
- **初始化优化**: 智能采样、密度加权等低风险改进
