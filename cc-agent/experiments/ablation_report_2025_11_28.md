## 核心洞察

### 1. X²-Gaussian (K-Planes) 是最稳定的增益技术

| 场景 | X 单独效果 | 最佳组合 | 最佳效果 | X 在最佳组合中 |
|------|-----------|----------|----------|---------------|
| Foot-3 | -0.09 dB | IX | +0.23 dB | ✅ |
| Chest-6 | +0.01 dB | XF | +0.09 dB | ✅ |
| Abdomen-9 | **+0.17 dB** | X | +0.17 dB | ✅ |

**结论**:
- **X 在所有最佳组合中都存在** - 证明 K-Planes 特征增强是核心技术
- X 在视角越多的场景效果越好（从 -0.09 到 +0.17）
- 在 9-views 场景，X 单独使用即为最优，无需组合

### 2. Init-PCD (I) 在极稀疏场景表现突出

| 场景 | I 单独效果 | I 在最佳组合中的作用 |
|------|-----------|---------------------|
| Foot-3 | -0.06 dB | **必需** (top-4 全含 I) |
| Chest-6 | -0.13 dB | 可选 (第2名含 I) |
| Abdomen-9 | -0.30 dB | **负面** |

**结论**:
- I 在 3-views 极稀疏场景是关键技术
- I 在密集场景有害，应禁用
- 需要场景自适应策略

### 3. 技术组合存在显著协同效应

| 场景 | 单技术效果之和 | 最佳组合效果 | 协同效应 |
|------|---------------|-------------|---------|
| Foot-3 (IX) | -0.06 + (-0.09) = -0.15 | +0.23 | **+0.38** |
| Chest-6 (XF) | +0.01 + (+0.01) = +0.02 | +0.09 | **+0.07** |
| Abdomen-9 (X) | +0.17 | +0.17 | 0 |

**结论**:
- **极稀疏场景 (3-views) 协同效应最强** (+0.38 dB)
- 随着视角增多，协同效应递减（密集场景无需组合）
- **全技术组合 (IXFB) 在所有场景都不是最优** - 技术冲突大于协同

### 4. Bino (B) 表现中性

| 场景 | B 单独效果 |
|------|-----------|
| Foot-3 | -0.21 dB |
| Chest-6 | -0.05 dB |
| Abdomen-9 | -0.00 dB |

**结论**:
- B 在极稀疏场景有害
- B 在密集场景无影响
- 考虑移除或仅在特定场景启用

```
if views <= 3:
    enable: Init-PCD, X²-Gaussian
    disable: FSGS, Bino
elif views <= 6:
    enable: X²-Gaussian, FSGS
    disable: Init-PCD, Bino
else:  # views >= 9
    enable: X²-Gaussian only
    disable: Init-PCD, FSGS, Bino
```

## 总结

### 实验规模
- **48 个完整实验** (16 配置 × 3 场景)
- **3 个场景**: Foot-3views / Chest-6views / Abdomen-9views
- **4 种技术**: Init-PCD (I) / X²-Gaussian (X) / FSGS (F) / Bino (B)
- **15 种技术组合** + 1 个 baseline

### 核心发现

1. **X²-Gaussian 是最稳定的核心技术**
   - 在所有 3 个场景的最佳配置中都存在
   - 视角越多，单独使用效果越好

2. **Init-PCD 在极稀疏场景关键，在密集场景有害**
   - 3-views: 必需组件（+0.23 dB with IX）
   - 9-views: 禁用（-0.30 dB 单独使用）

3. **技术组合有协同效应，但全组合不是最优**
   - 3-views 协同效应最强 (+0.38 dB)
   - 全技术 IXFB 在所有场景都不是最佳

4. **场景自适应策略是正确方向**
   - 不同视角数需要不同技术组合
   - 已验证的最优配置：3v→IX, 6v→XF, 9v→X
