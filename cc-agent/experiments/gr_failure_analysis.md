# GR-Gaussian 失败分析报告

**日期**: 2025-11-25
**项目**: R²-Gaussian
**技术**: GR-Gaussian (Graph-Regularized Gaussian Splatting)
**数据集**: Foot-3 views
**Baseline**: PSNR 28.487 dB, SSIM 0.9005

---

## 1. 实验结果汇总

| 实验版本 | PGA | 5k PSNR | 10k PSNR | 20k PSNR | 30k PSNR | 30k SSIM | vs Baseline |

## 2. 核心失败原因分析

### 2.1 PGA (Pixel-Graph-Aware Gradient Strategy) 实现问题

**现象**:
- `gr_pga_fixed_30k`: 性能随训练**持续恶化** (25.8 → 18.0 dB)
- `gr_pga_fixed_v2`: 修复后仍低于 baseline 0.46 dB

**PGA 算法原理**:
```
g_i += λ_g · Σ(Δρ_ij) / k
其中 Δρ_ij = ρ_j - ρ_i (邻居密度差异)
```

**问题分析**:
1. **梯度方向可能错误**: PGA 鼓励每个点向邻居密度的平均值靠近，但在稀疏视角下这可能破坏正确的密度分布
2. **与 densification 冲突**: 当 densification 改变点云结构时，图的拓扑关系失效，但 PGA 仍在使用旧的邻居信息
3. **累积效应**: 错误的梯度修改在长期训练中累积，导致模型崩溃

**代码位置**: `r2_gaussian/utils/graph_utils.py:193-261`

### 2.2 Graph Laplacian Loss 贡献极小

**现象**:
- 历史日志显示 `graph_loss` 全程在 `0.000000 ~ 0.000001` 范围
- 对总损失的贡献几乎为零

**公式**:
```
L_lap = λ_lap * Σ w_ij * (ρ_i - ρ_j)²
```

**问题分析**:
1. **密度差异本身很小**: 在 R²-Gaussian 中，高斯点密度经过 softplus 激活后分布紧凑，邻近点差异很小
2. **边权重衰减过快**: `w_ij = exp(-d²/σ²)` 对距离敏感，远距离邻居权重趋近于零
3. **λ_lap 值太小**: 即使提升 10 倍到 0.008，乘以极小的密度差异仍然贡献微弱

**代码位置**: `r2_gaussian/utils/loss_utils.py:106-168`

### 2.3 稀疏视角场景的特殊性

**GR-Gaussian 论文假设**:
- 原论文可能在更多视角（如 9-50 views）下设计
- 在较密集视角下，图正则化有助于减少伪影

**3 视角的困境**:
1. **信息极度稀缺**: 只有 3 个训练视角，模型本身就难以学习完整的 3D 结构
2. **图约束可能有害**: 强制邻近点密度平滑可能抹杀正确的边缘和细节
3. **泛化能力关键**: 在 50 个测试视角中，大部分远离训练视角，需要强泛化而非强约束

## 3. 各组件独立评估

| 组件 | 设计目标 | 实际效果 | 评估 |
|------|---------|----------|------|
| **Graph Laplacian Loss** | 平滑邻近点密度 | 损失值过小，几乎无效 | ⚠️ 无效 |
| **PGA** | 引导梯度考虑邻居关系 | 破坏收敛，性能下降 | ❌ 有害 |

| **图更新策略** | 跟踪点云变化 | 实现正确但无法弥补上述问题 | ⚠️ 中性 |

## 4. 历史问题追溯

### 4.1 早期实验设计问题
- **混合实验**: 最初将 CoR-GS (双模型) 和 GR-Gaussian 混合，无法公平对比
- **参数设置**: 最初 λ_lap=0.0008 太小，graph_update_interval=1000 太大

### 4.2 修复尝试记录
| 修复版本 | 修复内容 | 结果 |
|---------|---------|------|
| v1 | 基础实现 | 失败 |
| v2 | 提升 λ_lap 10倍，降低 interval | 仍低于baseline |
| 方案B | _prune_optimizer 保留梯度 | 无效 |
| 方案C | PGA 时序调整 | 性能暴跌 |
| pga_fixed_v2 | 修正 PGA 作用对象 | 仍低于baseline 0.46dB |

## 5. 根本原因总结

### 5.1 算法层面
1. **GR-Gaussian 不适合超稀疏场景**: 论文方法可能需要更多视角才能发挥作用
2. **图正则化与表达能力冲突**: 在信息极度稀缺时，约束可能限制模型学习能力
3. **PGA 原理在 CT 重建中不适用**: 医学 CT 的密度分布与自然图像不同

### 5.2 实现层面
1. **Graph Laplacian loss 设计不合理**: 公式和参数无法产生有意义的约束
2. **PGA 与 densification 不协调**: 图结构变化时梯度修改策略失效
3. **缺乏自适应机制**: 固定参数无法适应不同训练阶段

---

## 6. 建议行动

### 6.1 短期（放弃 GR-Gaussian）
- **结论**: 在 3-view 场景下，GR-Gaussian 技术不适用
- **建议**: 转向其他更有潜力的技术（如 X²-Gaussian、SSS 等）

### 6.2 中期（如果仍想探索）
- 只使用 Graph Laplacian，完全禁用 PGA
- 尝试更激进的 λ_lap（如 0.1~1.0），观察是否有效果
- 仅在 densification 结束后启用图正则化

### 6.3 长期（如果要发论文）
- 设计适合超稀疏场景的图正则化方法
- 考虑多尺度图结构
- 引入自适应权重机制

## 8. 最终结论

**GR-Gaussian 在 R²-Gaussian 的 3 视角稀疏场景下无法提升性能**。

核心原因是:
1. 图正则化在信息极度稀缺时反而限制了模型表达能力
2. PGA 策略破坏了正常的梯度下降过程
3. 算法设计可能针对更多视角的场景

**建议**: 放弃 GR-Gaussian，专注于其他更有潜力的技术方向。
